§ 03

使用指南

什么内容用什么主题

按内容场景推荐最适合的主题。两个原则：(1) 不确定就选 warm-brass，它是默认； (2) 暖色 Editorial 优先用于产品摄影和故事叙述，Swiss 优先用于工具型 / 数据型 / 列表型内容。

内容场景 推荐主题 风格 理由
小红书产品摄影 · 主推 / 新品 warm-brass Editorial 默认 RV 招牌色，呼应金箔/黄铜饰品。米色纸配手部产品图最自然。
礼盒攻略 · 按送礼对象分组 rv-gold 瑞士风 Swiss Grid 列表最适合瑞士排版，结构清晰，CTA 醒目。
工艺幕后 · artisan / small-batch warm-brass / burnished-brass Editorial 工序、手部、工作台素材最适合暖调 Editorial。burnished-brass 增加历史厚重感。
材质科普 · 实心金箔 vs 镀金 stone-blue Editorial 冷色 Editorial 给科普文字一种可信感，不喧宾夺主。
半宝石 / 自然系新品 olive-stone / sage 任一 olive-stone 给 Editorial，sage 给瑞士风。色彩与素材自身呼应。
5 套穿搭 / "5 种戴法" ivory Editorial 最浅奶油让产品图最亮，不抢视觉。
FAQ / 客户问答 stone-blue / rv-gold Swiss 信息密度高的内容用瑞士风更整齐。
客户证言矩阵 rv-gold Swiss 用 S12 Matrix 排版，rv-gold 给温暖可信感。
Q4 礼盒 / 圣诞倒计时 honey Swiss 节日能量，温暖而不俗气。
限量发售 / 即将售罄 brick Swiss 陶土红 = 紧急 + 温暖（不是工业红）。
情人节 / 母亲节 / 周年纪念 honey Editorial / Swiss 节庆色，但保持 RV 暖中性，不过分甜腻。
暗调摄影 · 烛光 / 丝绒背景 midnight-brass Editorial 唯一暗调。原素材真暗时才用。
公众号 21:9 + 1:1 主封面 warm-brass / rv-gold 任一 封面图用 Editorial 内页，瑞士封面用瑞士风。
公众号文章内页图 ivory / rv-gold 任一 正文图保持轻、净，不抢文章标题。
不确定时 warm-brass Editorial 默认。80% 场景都能用。

§ 06

使用规则 · 避坑清单

来自 RV 品牌调性的硬性规则

这些是 Rare Visual 品牌调性 + 上游 Guizang 视觉规范合并的硬性规则。 违反任意一条 = 视觉失控。

1. 越大越细

Editorial 大字反而用更细 weight（500 而非 900）。Swiss display 必须 ≤ 300。绝对禁止 Editorial 标题用 700+ weight 配 124px+ size。

2. 一个 accent

Swiss 只能用一个 accent。绝对不要混 rv-gold + honey 同时出现在一个 deck 里。同理 Editorial accent 只用一次：section marker / page number / pull quote rule。

3. 永远不要纯黑纯白

背景用 #faf6ee（象牙白）而非 #ffffff；墨色用 #1a1612 而非 #000000。暗夜黄铜用 #14110d 而非 #000000。纯值会破坏层次感。

4. 字号下限 22-28px

在 1080×1440 卡上，最小可读字号 = 22-28px。Swiss .t-meta ≥ 18px。低于这个，mobile 下采样就糊了。Validator R4 会强制检查。

5. Editorial 一套卡一个主题

不要在同一 deck 里混 warm-brass 和 stone-blue 的 accent。只能用不同 .paper / .accent-soft 做 variation。Validator 会检测 accent 越界。

6. 暗夜黄铜限制

这是唯一的暗调 Editorial。仅用于源素材已经暗 + 暖的场景。Hero cover、节日 drop 可用，其他一律用 warm-brass。

7. 不要 AI 紫 / 霓虹绿

绝对禁止 #A855F7（AI 紫）、#00FF85（霓虹绿）、chrome-bling 风格。Rare Visual 调性是 intentional, warm, grounded，不要纯数字炫技色。

8. 标题最多 2 行

.h-display 在 desktop 最多 2 行。超出就改用 .h-xl（88px）而非加大字号。Hero 区域内文 ≤ 4 行。

9. 卡片填充 ≥ 75%

3:4 卡的内容（文字 + 图 + 数据）必须覆盖 ≥ 75% 画布高度。任何 >15% 画布高度的纯空白都需要"留白理由"：hero image 呼吸 / 单句宣言 / 段落前后 whitespace。

10. Editorial 必须有氛围层

不能纯色 cream 背景加文字就交付。必须有 WebGL ink-flow canvas 或 paper grain overlay（grain opacity .35, multiply）。杂志感来自叠层。

11. 不要堆砌 emoji

Rare Visual 调性是 curated / intentional，emoji 会破坏。可用 Lucide icons，但每个组件最多 1-2 个。

12. 不要伪造数据

"92% 用户满意"这种伪造数据禁止。如果要 placeholder，写 "SAMPLE" 或用 [X%] 标识。客户引言用 "verified buyer" 或 "user-submitted"，但必须有真人出处。
